一、森林走訪的基本概念
- 森林（Forest）：由多棵樹（Tree）組成的集合。
- 走訪順序（Traversal Order）：常見有前序（Preorder）、中序（Inorder）、後序（Postorder）。

二、森林轉樹的走訪策略
- 森林的走訪順序可以透過「轉成二元樹（Binary Tree）」後再進行走訪。
- 森林的中序 = 將森林轉成二元樹後的中序
- 森林的後序 = 將森林轉成二元樹後的後序

```text

範例一:
        A          F      H   
     /  |  \       |    /    \
    B   C    D     G   J      K

Output: E B C D G J K I H F A

範例二:
        A               G          J
     /  |  \          /    \     /   \
    B   C    D       H      I   K     L
       / \                      \
      E   F                      M

Output: B E F C D H I M K L J G A 

```

三、B.T轉森林/樹的走訪策略
- 根的右兒子往上拉，成為根的兄弟。
- 刪除根的兄弟的link
- 將每顆樹轉B.T，
  - 將右兒子逆時針轉45度
  - 將父子的Link加上
  - 將sibling的link刪除

```text

範例一:

原始樹:

            A
           / \
          B   C
         / \   \
        D   E   G
       /     \
      K       F
               \
                H
                 \
                  I
                   \
                    J


拆解後的左子樹（中左）:

        A
        |
        B
       / \
      D   E
     /     \
    K       L


拆解後的右子樹（中右）:

        C
         \
          G
         /
        H
         \
          I
           \
            J


中央整理後 :

        A
        |

            A
          /   \
         B     C
        / \   / \

      /     \     \
```

四、為什麼要學會森林轉樹的轉換？

1. 統一資料結構，便於走訪與操作
  - 森林（Forest）由多棵樹組成，結構較複雜。
  - 將森林轉成「二元樹」後，可以用標準的前序、中序、後序走訪方式處理。
  - 這讓你可以用一套演算法處理所有樹，提升效率與一致性。
2. 便於儲存與序列化
  - 在記憶體或檔案儲存中，樹結構常需轉成線性格式（如陣列、字串）。
  - 若能轉成二元樹，就能用堆疊、佇列等結構輕鬆儲存與還原。
3. 支援複雜的操作與重構
  - 例如：刪除節點、合併子樹、重新排序兄弟節點等。
  - 若能掌握「右兄左子」的轉換邏輯，就能靈活地重構整棵樹或森林。
4. 應用於實際問題建模
  - 許多真實世界問題（如公司組織、人類家譜、檔案系統）都可建模為森林。
  - 若能轉成樹，就能用演算法進行搜尋、分析、視覺化等操作。
